# AI Free: мозги как Obsidian + skills как Codex/Claude

Документ фиксирует рабочую архитектуру, как превратить AI Free из мульти-провайдерного чата/код-агента в полноценный продукт с долговременной памятью, skill-системой и агентными режимами.

## 1. Что уже есть в проекте

По текущей структуре проект уже достаточно сильный:

- `src/providers/*` — провайдеры DeepSeek, Qwen, ChatGPT.
- `src/code-agent/*` — автономный coding agent с tool-call JSON-протоколом.
- `src/window-app/*` — локальное desktop/web окно на `127.0.0.1:4317`.
- `api/*` — OpenAI-compatible API.
- `src/agent-runtime/roles.mjs` — роли агентов.
- `state` — сохранение чатов, настроек, workspace, pipeline.
- `pipelineMode` — уже есть база для цепочек агентов.
- `coderMode` / `hardwareMode` — уже есть пер-чатовые агентные режимы.

Главный вывод: не надо переписывать проект. Надо добавить поверх существующей архитектуры 3 слоя:

1. Memory Core — долговременная память.
2. Skill Runtime — устанавливаемые скиллы.
3. Agent Orchestrator — мозг, который выбирает память, скиллы, роль, провайдера и инструменты.

---

## 2. Главная идея продукта

AI Free должен стать не просто бесплатным доступом к моделям, а локальным AI-рабочим пространством:

> Пользователь пишет задачу, а AI Free сам понимает контекст проекта, вспоминает прошлые решения, подключает нужные скиллы, выбирает подходящего провайдера и выполняет задачу через безопасные инструменты.

Позиционирование:

- Не просто чат.
- Не просто VS Code-плагин.
- Не просто OpenAI-compatible proxy.
- А локальный AI-секретарь/агент для проектов, кода, заметок, файлов и рутины.

### 3.5. Асинхронная память

Очень важный момент: память нельзя писать прямо в основной ответ модели. Иначе будет медленно и раздражающе.

Правильный поток:

1. Пользователь общается с агентом.
2. Ответ показывается сразу.
3. После ответа запускается фоновый memory-extractor.
4. Extractor решает:
   - что сохранить;
   - что обновить;
   - какие связи создать;
   - что не сохранять.
5. UI может показать маленькое событие: `🧠 Память обновлена: 2 факта, 1 решение`.

- `markdown.mjs` — чтение/запись Markdown с frontmatter.

- `extractor.mjs` — вытаскивает факты/решения из диалогов.

## 8. Безопасность

Skills могут быть опасными, если им разрешить выполнять скрипты. Поэтому:

### MVP

- skill = prompt + metadata;
- никаких автоматических shell scripts;
- команды идут только через существующий whitelist;
- skill может только попросить разрешения, но не обойти настройки.

### Позже

- signed skills;
- sandbox;
- проверка manifest;
- warning для high-risk permissions.

## 9. Реальный MVP на 3 этапа

### Этап 1 — Memory MVP

Цель: локальная память без векторов.

Сделать:

- `src/memory/store.mjs`;
- Markdown-хранилище;
- SQLite FTS, если добавим зависимость, либо JSON index без зависимостей;
- API:
  - `GET /api/memory/search?q=`
  - `POST /api/memory`
  - `DELETE /api/memory/:id`
- ручная кнопка `Save to memory`;
- автоматическое резюме диалога после ответа.

Первый MVP можно сделать вообще без новых npm-зависимостей: Markdown + JSON index. Это быстрее и проще для пользователей.

### Этап 2 — Built-in Skills

Сделать:

- `src/skills/builtins/*`;
- `skill.json` + `prompt.md`;
- загрузчик skills;
- `/skill <id> <task>`;
- выбор skill в UI;
- подмешивание skill prompt в `createCodeSystemPrompt()`.

### Этап 3 — Orchestrator

Сделать:

- auto skill matching;
- memory context builder;
- project context summary;
- per-project memory vault;
- pipeline + skills integration.

## 10. Что НЕ делать сейчас

Не советую сразу делать:

- Elasticsearch;
- полноценный graph database;
- маркетплейс skills;
- удалённую установку skills из GitHub без проверки;
- сложный UI как Obsidian Canvas;
- тяжёлые embedding-модели как обязательную зависимость.

Это всё может убить простоту установки. Сначала надо сделать локальную память и skills без боли.

Так пользователь сразу почувствует “мозги”: агент будет помнить, что уже делали в проекте.

## 12. Продуктовая формулировка

Можно описывать фичу так:

> AI Free Memory — локальная база знаний для ваших проектов. Агент запоминает решения, ошибки, команды, структуру проекта и ваши предпочтения. Всё хранится у вас на компьютере в читаемом Markdown-формате, как личный Obsidian для AI.

А skills:

> AI Free Skills — устанавливаемые навыки агента: code review, bug fix, тесты, документация, Telegram-боты, VPN-инфраструктура, релизные посты и другие сценарии. Skill добавляет агенту инструкции, правила и рабочий процесс под конкретную задачу.
